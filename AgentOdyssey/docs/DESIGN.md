## 一、项目定位

AgentOdyssey 是一个**互动式 AI Agent 学习游戏**，用太空冒险的方式系统讲解 AI Agent 开发。

**核心特点：**
- 🎮 游戏化：故事驱动、星球收集、星星奖励
- 📚 系统化：从 LLM 基础到生产级 Multi-Agent 系统，完整路径
- 🎯 分级：同一内容三种难度，小学生到架构师都能学
- 💻 纯前端：HTML + CSS + JavaScript，无需构建工具，打开即用
- 🔬 真实案例：帝国篇基于真实开源项目 [EDICT](https://github.com/cft0808/edict)

## 二、技术架构

### 文件结构

```
AgentOdyssey/
├── index.html                  # 主页面（4 个屏幕：标题/地图/关卡/胜利）
├── css/
│   └── styles.css              # 全局样式（太空主题、难度颜色、响应式）
├── js/
│   ├── game-state.js           # 全局状态（stars、completed、currentDifficulty）
│   ├── renderer.js             # 渲染引擎（关卡内容渲染、难度切换器）
│   ├── interactions.js         # 交互逻辑（quiz 判题、动画、星星收集）
│   ├── main.js                 # 入口（星空背景、路由、地图渲染）
│   └── planets/                # 关卡数据（每关一个文件）
│       ├── p1-llm.js
│       ├── ...
│       └── p21-empire-5.js
└── docs/
    ├── DESIGN.md               # 本文件：项目总体设计
    └── empire-series-design.md # 帝国篇详细设计（含 p17-p21 关卡规划）
```

### 关卡数据格式

每个关卡文件向全局 `PLANETS` 数组 push 一个对象：

```javascript
PLANETS.push({
  id: 'p1',          // 唯一 ID
  icon: '🌍',        // 地图显示图标
  num: '星球 1',     // 序号标签
  name: 'LLM 星',   // 关卡名称
  desc: '...',       // 简短描述

  difficulties: {
    easy: { sections: [...] },   // 🟢 简单模式
    hard: { sections: [...] },   // 🟡 困难模式
    hell: { sections: [...] },   // 🔴 地狱模式（部分关卡有）
  }
});
```

### Section 类型

| 类型 | 用途 | 关键字段 |
|------|------|---------|
| `story` | 对话故事推进剧情 | `html` |
| `concept` | 概念讲解卡片 | `title`, `html` |
| `code` | 代码示例 + 解释 | `title`, `code`, `explanation` |
| `pitfalls` | 常见坑点列表 | `title`, `items[]` |
| `quiz` | 单选测验 | `q`, `opts[]`, `ans`, `feedback_ok`, `feedback_err` |

### 难度系统

- **切换器**：关卡顶部实时切换，无需重新加载
- **地狱模式**：`renderer.js` 动态检测 `planet.difficulties.hell` 是否存在，存在则解锁
- **进度追踪**：每个难度独立记录完成状态，星星分别计算

## 五、帝国篇设计理念

详细设计见 [`empire-series-design.md`](./empire-series-design.md)。

**核心思想：** 1300 年前唐朝用三省六部制解决"如何让一个大帝国有序运转"，今天我们用同样的架构思想解决"如何让 12 个 AI Agent 可信协作"。

**五层架构对应关系：**

```
P17 架构层 → 三省六部分工   → 解决"谁做什么"
P18 控制层 → 状态机+权限矩阵 → 解决"怎么保证按规矩来"
P19 观测层 → Event-Driven   → 解决"怎么知道在发生什么"
P20 质量层 → 门下省审核     → 解决"怎么保证输出质量"
P21 工程层 → 成本+可靠性    → 解决"怎么真正上线"
```

**与主流框架对比：**

| 特性 | CrewAI | AutoGen | 三省六部（EDICT） |
|------|--------|---------|----------------|
| 强制审核 | ❌ | ⚠️ 可选 | ✅ 门下省必经 |
| 实时看板 | ❌ | ❌ | ✅ WebSocket 推送 |
| 任务干预 | ❌ | ❌ | ✅ 随时叫停/恢复 |
| 权限矩阵 | ❌ | ❌ | ✅ 白名单强制检查 |
| 流转审计 | ⚠️ | ❌ | ✅ Append-only log |
