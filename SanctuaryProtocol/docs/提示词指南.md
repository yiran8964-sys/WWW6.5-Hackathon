建议使用的 AI 工具：Cursor

复制以下提示词发送给 AI：

本项目是一个“去中心化互助与隐私保护协议”。它包含资金捐赠端（藏家模式）和资金拨付端（弱势群体通过心理觉察生成签名领钱）。

【核心技术需求】
1. 采用 Solidity ^0.8.0 版本。引入 OpenZeppelin 的 `Ownable`, `ReentrancyGuard`, 和 `ECDSA` 库。
2. 基础数据结构（情绪上链）：
   - `MemoryRecord` 结构体：包含图卡ID数组、字卡ID数组、前端加密后的日记密文、存入时间。
   - 保留 `depositMemory` 和获取时间的 `timeOffset` 演示逻辑。
3. 资金端 (Input / 捐赠机制)：
   - 编写 `donateAndMint(uint256 cardId) external payable` 函数。
   - 资金留在合约中。触发事件 `DonorAppreciated(address indexed donor, uint256 cardId, uint256 amount)`。
4. 反女巫防线 (Anti-Sybil)：
   - 定义 `mapping(address => bool) public isWhitelisted;`，并提供管理员添加白名单的函数。
   - 定义 `mapping(address => bool) public hasClaimed;` 防止重复领取。
5. 受助端 (Output / 签名验证拨付)：
   - 定义 `address public verifierSigner;`（用于验证前端发放的达标签名）。
   - 编写 `claimAid(bytes memory signature) external nonReentrant` 函数。
   - 验证流程：
     a) 要求调用者 `isWhitelisted[msg.sender] == true`。
     b) 要求 `!hasClaimed[msg.sender]`。
     c) 利用 ECDSA 验证 signature。该签名的明文应为 `keccak256(abi.encodePacked(msg.sender))`，即验证“该用户的地址是否确实被 verifierSigner 签发了通行证”。
   - 执行流程：验证通过后，将 `hasClaimed` 置为 true，并向 `msg.sender` 转账固定金额（如 0.01 ether）。

请输出完整、高安全性的合约代码，并附带详尽注释。

## 步骤 1：智能合约开发 (Smart Contract)
建议使用的 AI 工具：Cursor / ChatGPT / Claude 目标：一次性写出安全、符合逻辑的 Solidity 合约代码。
复制以下提示词发送给 AI：
你现在是一位资深的 Web3/Solidity 开发工程师。请根据以下需求，为我编写一个名为 `OHCardSanctuary.sol` 的智能合约。

【项目背景】
这是一个主打“情绪疗愈”的 Web3 OH卡系统。用户可以在前端挑选植物图卡和字卡，写下加密日记并存入区块链，随着时间推移，植物会发生状态演变。

【核心技术需求】
1. 采用 Solidity ^0.8.0 版本。使用 OpenZeppelin 的 Ownable 合约。
2. 数据结构：
   - 定义 `MemoryRecord` 结构体，包含：`uint256[] imageIds` (选中的图卡绝对ID)、`uint256[] wordIds` (选中的字卡ID)、`string encryptedJournal` (前端加密后的密文)、`uint256 timestamp` (存入时间)。
   - 定义 `mapping(address => MemoryRecord[]) private userMemories;`
3. 核心写入功能 `depositMemory`：
   - 接收 `imageIds`, `wordIds`, `encryptedJournal`。
   - 记录当前的区块时间戳。
   - 触发事件 `CardResonated(uint256 cardId)`，遍历传入的图卡和字卡ID触发此事件（用于前端统计共鸣次数）。
4. 核心读取功能 `getMyMemories`：
   - 仅限 `msg.sender` 读取自己地址下的所有记录。
5. 辅助功能（伪随机）：
   - 编写一个 `getRandomCard(uint256 max)` 函数，利用 `keccak256(abi.encodePacked(block.timestamp, msg.sender, block.difficulty))` 生成一个介于 0 到 max 之间的伪随机数。
6. 测试演示功能（时间机器）：
   - 定义全局变量 `uint256 public timeOffset = 0;`。
   - 编写 `setTimeOffset(uint256 _offset) external onlyOwner` 函数。
   - 修改合约中获取时间的逻辑：内部获取时间戳时，请返回 `block.timestamp + timeOffset`，以便于管理员在测试时手动拨快时间。

请输出完整、带有详细注释的合约代码，并确保没有重入漏洞和编译警告。

## 步骤 4：时间演示控制台与演化算法 (Time Machine Setup)

2. 点击时调用智能合约的 `setTimeOffset` 方法。

【核心演化算法 (Hooks)】
在“我的庇护所”渲染记录时，需要根据区块链当前时间与存入时间的差值，计算植物的演化。

- 每经过 1 天（86400秒），阶段数 + 1。
- 公式：`展示阶段 = Math.min(5, 存入时的原始阶段 + 经过的天数)`。
- 请利用 `cardConfig.ts` 中的字典，根据当前计算出的 `展示阶段` 和原始的 `轨迹ID`，去反查出对应的图片路径并渲染在界面上。
- 配上一句温柔的提示，例如：“它已经从 [创伤] 生长为了 [保护隔离]”。

请提供控制台组件代码以及这个用于计算当前生命阶段和获取对应图片的 Hook 函数。
