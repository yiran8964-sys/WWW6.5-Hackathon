产品需求文档 (PRD)：Web3 庇护所互助协议 & OH 卡前端系统

版本: V0.2 (协议化升级版) | 项目代号: Sanctuary (庇护所)

1. 引言 / 概述 (Introduction / Overview)

本项目不仅是一个心理疗愈应用，更是一个基于以太坊/Solidity 智能合约开发的去中心化互助与隐私保护协议。
在传统医疗众筹（如水滴筹）中，边缘群体的求助往往伴随着“自证弱者”的隐私剥夺与病理化审查。本项目旨在利用 Web3 技术重构这一权力结构：

底层引擎（协议）：一个双边经济资金分配插件。它通过智能合约锁定捐赠资金，并接收特定格式的证明（MVP阶段为签名凭证，未来为零知识证明 ZKP）来触发无审查的自动打款。该协议可复用于任何具备“双边进出流转”性质的前端工具中。

首发前端壳子（OH卡系统）：一个去金融化、本地优先的植物主题心理投射工具。它将传统的“提交病历证明”转化为“完成心理觉察（工作量证明 PoW）”。

2. 目标 / 目的 (Goals / Objectives)

协议层目标：实现“捐赠端（Input）”与“受助端（Output）”的无信任资金闭环。藏家买单艺术属性，资金进入金库；求助者通过心理觉察解锁资金。

应用层目标：实现 4 种 OH 卡牌阵、前端绝对私密的情绪封存，以及基于 6 大心理轨迹的时间流变视觉演化。

反女巫与安全目标（MVP）：通过“高信任邀请码（白名单）”和“前端前端行为验证+签名”的组合，在不引入复杂 ZKP 电路的前提下，防范脚本刷量，保护资金池。

3. 目标受众 / 用户画像 (Target Audience / User Personas)

资金端 (Input) - 庇护所守护者/藏家：愿意为社会公益与数字艺术买单的用户。他们支付稳定币/ETH 收藏具有心理学隐喻的植物 OH 卡，资金进入协议金库。

受助端 (Output) - 边缘群体/求助者：面临心理危机或生存压力的群体。在“去病理化”的界面中完成自我探索，无需暴露隐私即可获得资金支持。

治理端 - NGO/信任节点：在现实中接触真实求助者的机构，负责发放“创世邀请码（上链白名单）”以阻断女巫攻击。

4. 双边用户故事 / 使用场景 (User Stories)

故事 1：捐赠与数字收藏（资金蓄水）

作为一名藏家，我进入“守护者模式”，浏览 30 张植物生命轨迹卡。我支付 0.05 ETH 收藏了《创伤》这张卡片，我的资金进入了庇护所金库，且我的善举（地址）被永久记录在链上。

故事 2：本地化的心理觉察（生成 PoW）

作为一名求助者（拥有邀请码），我进入“疗愈者模式”。我选择牌阵，挑选图文卡，并专注地写下了 50 字以上的日记，停留了超过 3 分钟。我的日记在前端本地被加密，甚至可选择阅后即焚（物理清除）。

故事 3：无审查的尊严救助（资金释放）

作为一名求助者，当我完成上述觉察后，前端静默判定我达标，并利用内置密钥生成了一份“已完成探索”的密码学签名（模拟 ZKP）。我点击“领取疗愈基金”，合约验证签名与我的白名单身份后，自动且毫无情绪地向我打款（如 0.01 ETH），全程无人知晓我写了什么。

5. 功能需求 (Functional Requirements)

5.1 前端页面规划 (Frontend Modules)

双边模式切换：首页提供 [🌿 疗愈者模式]（核心工具壳子）和 [💎 守护者模式]（资金端入口）的切换。

守护者画廊 (Sponsor Mode)：展示 30 张 OH 卡的艺术意象，附带 [捐赠并收藏] 按钮（调用合约 donateAndMint）。

疗愈者工具 (Healer Mode)：

静态资产映射：6 大生命轨迹，共 30 张图卡与字卡。

图文解析引擎：基于字典配置，在页面展示柔和的专业解析脚手架。

私密书写区：钱包签名授权的 AES 前端加密。

工作量证明与领款组件 (PoW & Claim Engine)：

行为监听：记录页面停留时间（需 > 3分钟）和 Textarea 字数（需 > 50字）。

凭证生成：达标后触发弹窗，利用预置在环境变量中的“Admin 私钥”对用户的钱包地址进行数字签名。

领款交互：展示 [领取基金] 按钮，调用合约的 claimAid 方法。

5.2 智能合约底层协议 (Smart Contract Engine)

合约名称：SanctuaryProtocol.sol

资金托管与捐赠 (Input)：

donateAndMint(uint256 cardId) payable：接收资金进入合约池，触发 DonorAppreciated 事件。

白名单与防女巫 (Anti-Sybil)：

mapping(address => bool) public isWhitelisted;（仅限管理员添加真实受助者）。

mapping(address => bool) public hasClaimed;（每个地址限领一次）。

零知识证明模拟与资金拨付 (Output)：

claimAid(bytes memory signature)：接收前端生成的签名。

验证逻辑：1. 使用 ECDSA 恢复出签名者是否为官方 Admin；2. 检查调用者是否在白名单；3. 检查是否未曾领取过。

执行逻辑：条件满足后，自动向 msg.sender 转账固定额度的 ETH/代币，并将 hasClaimed 设为 true。

时间流变机制：保留 timeOffset 和 depositMemory 接口，支撑 OH 卡工具的本体功能。

6. 非功能需求 & 设计考量 (NFR & Design)

可组合性 (Composability)：合约资金池的 claimAid 接口设计必须解耦，未来除了 OH 卡，其他如“房树人涂鸦应用”或“创作者代码库多签”均可作为前端壳子接入此验证接口。

安全性：合约必须包含重入锁（nonReentrant），防止黑客在领款时利用 Fallback 漏洞抽干资金池。

视觉语言：低饱和度大地色系。在守护者模式突出“共建与荣光”，在疗愈者模式突出“隐私、抱持与安全感”。

7. 架构演进与未来规划 (Future Considerations)

V0.1 (当前 MVP)：使用“高信任邀请码 + 前端 Admin 私钥签名”模拟 ZKP，实现闭环。

V1.0 (去信任化)：引入 ZK-Email，受助者在本地生成邮箱持有证明发给合约，彻底抛弃中心化白名单。

V2.0 (全链双边经济)：捐赠者获得的卡牌升级为带版税的动态 NFT，二级市场的每次转手都自动抽取 10% 重新反哺流入协议救助资金池。

Web3 去中心化互助与隐私保护协议 (项目代号：Sanctuary/庇护所)

版本: V0.1 (概念验证阶段)
核心定位: 一个基于零知识证明与双边经济模型的去中心化资源流转插件/底层协议，致力于在无审查、去病理化的前提下，为边缘群体及个体化需求提供资金支持。

0. 项目愿景与破局点

传统医疗众筹（如水滴筹）与NGO组织在救助边缘群体（特别是心理危机群体）时，往往陷入“自证弱者”的权力结构：求助者必须让渡极度隐私、接受病理化审查才能获得资源。

本项目旨在利用 Web3 技术（智能合约 + 零知识证明）重构这一过程，将“证明我有病”转化为“证明我完成了自我觉察”，将“平台施舍”转化为“代码协议自动流转”，将单向捐赠转化为“双边经济价值交换”。

1. 核心模块：心理自助工具的互助资金流转

这是本协议的第一个，也是最具社会价值的“前端环境壳子”。

1.1 本地优先 (Local-First) 的安全容器

物理隔绝监控： 心理自助工具（如OH卡图卡探索、自由书写、房树人涂鸦）采用 PWA 纯前端架构。用户断网亦可使用。

数据零上云： 用户的觉察日志、绘画轨迹仅保存在设备本地存储（localStorage）。关闭页面或点击“销毁”后，数据彻底物理清除，从根本上杜绝第三方审查和数据外泄。

去病理化体验： 界面剔除临床诊断词汇，采用隐喻、原型等心理学框架，为边缘群体提供极具“抱持感”的容器。

1.2 无主权资金池的自动拨付

稳定币金库： 在以太坊 L2 网络部署智能合约资金池，仅接收 USDC/USDT 等稳定币，避免受助者承担加密市场波动风险。

零知识证明提取： 当用户在心理自助工具中完成了特定的探索流程后，前端代码在设备本地生成密码学证明（ZKP），该证明向智能合约宣告“该匿名用户真实完成了互助练习”，但不透露任何具体内容或个人身份。

代码触发放款： 智能合约验证 ZKP 有效且防重放（防一证多刷）后，自动且毫无情绪地向该用户的匿名钱包地址放款微小额度的生存基金/咨询基金。

2. 扩展模块：独立于工具的 Web3 水滴筹与个体化众筹底层协议

这个项目不仅仅是“一个心理求助网站”，它本质上是一个“双边经济资金分配插件（Protocol）”。

我们将其抽象为核心引擎和多个前端场景：

2.1 引擎的三大职能

资金托管： 智能合约锁定捐赠池。

介质发行： 发行作为交换价值的数字资产（NFT 等）。

验证路由： 接收多种格式的证明（ZKP、多签、投票），判定有效并触发打款。

2.2 作为“双边经济插件”的终极优势

这赋予了协议无穷的生命力（Composability/可组合性）：如果你开发了一个心理自助页面，你可以接入这个资金池；如果有独立创作者在其他网站做艺术展览、或者遭受躯体危机的边缘群体建立了自己的求助页面，只要他们通过了特定的验证接口，就都能调用这同一个去中心化资金池。这是一种泛用的底层公共物品。

3. 双边经济机制及其进出示例（核心设计）

该机制彻底解决了传统捐赠单向失血、缺乏共识、无法自造血的问题。

在每一个具体的“前端壳子”中，都有资金端的输入（Input）和受助端的输出（Output），并在中间产生流转介质。

3.1 示例一：【心理互助环境】（本期主打）

这是目前最核心的双边经济模型。

进（Input）/捐赠端： 赞助者使用稳定币向智能合约购买**“心理图卡（NFT）”**。稳定币进入智能合约金库，捐赠者获得兼具艺术价值和行善凭证的数字卡牌。

出（Output）/受助端： 边缘群体用户在“本地心理自助工具”中，使用这些被捐赠购买下来的“心理图卡”进行OH卡自我觉察。完成交互后，前端生成零知识证明（ZKP），用户凭此向金库申请等额稳定币救助。

闭环总结： 捐赠者买单的艺术品，恰恰成为了求助者探索心理、触发救助款发放的工具。资金与工具形成了完美的价值互文。此图卡甚至可以设定版税回流（Secondary Sales Royalty），每次在捐赠者之间转手，都有资金自动补充进互助池。

3.2 示例二：【独立创作者环境】（未来扩展）

此机制同样可以横向扩展。

进（Input）/捐赠端： 天使赞助人购买一个项目概念的“创世徽章”，资金进入引擎锁定。

出（Output）/受助端： 独立创作者将自己的 Demo 或代码库哈希上传至网络，由信誉节点（担保人）多重签名确认真实进度。引擎验证多签有效，将资金拨付给创作者。

闭环总结： 流转的介质是“创新的交付与共识的凭证”。

4. 隐私安全与防女巫攻击 (Anti-Sybil) 策略

纯粹匿名的互助资金池面临最大的生死挑战是“女巫攻击”（黑客通过自动化脚本伪造海量心理自助行为来榨干资金）。为保护边缘群体无需提交“病历实名”，我们需要用技术摩擦换取安全性：

4.1 当前 MVP 阶段 (第一版本) 的妥协策略

为了快速验证最小闭环，MVP 阶段不应陷入复杂的抗审查军备竞赛。

高信任白名单与邀请制： 智能合约仅对拥有特定“创世邀请码”的用户开放提取权限。邀请码由现实中具有高信任度的核心节点或NGO工作者定向、私下发放给真正需要的群体。

金额熔断机制： 设定合约硬性规则（如每个地址每日限领 50 U，整个资金池每日最大流出限额），防止被一次性抽干。

4.2 未来升级阶段的防作恶机制探索

当产品跑通后，逐渐引入更加“去信任化 (Trustless)”的机制：

ZK-Email (零知识证明邮件验证)： 用户在本地证明自己拥有一个长期活跃的（如 2023 年前注册的）Gmail 或网易邮箱，生成证明发给智能合约。合约只知道“这是一个真实拥有历史邮箱的人类”，但绝不知道邮箱具体地址是什么。极高地提升了脚本的作恶门槛。

可选的链上实名 (DID)： 用户可选是否向特定可信审查节点提供病历，审查节点人工核实后，不公布隐私，仅为该用户发放不可转让的灵魂绑定代币 (SBT)。该 SBT 代表“极度危机”，持有者提取资金的限额或速度将获得提升。

前端经济摩擦 (工作量证明 PoW)： 在用户使用心理工具的二十分钟内，前端静默进行高强度的哈希运算（挖矿）。真实用户只消耗一点手机电量，而试图并发一万个脚本的黑客则需要支付昂贵的服务器算力成本，迫使其在经济层面停止作恶。
