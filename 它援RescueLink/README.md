# 🐾 它援 RescueLink

**一个连接真实救助需求、救助伙伴与社会支持资源的互助支持与可信记录工具。**

它援 RescueLink 面向流浪动物救助场景，试图解决真实救助中最常见、也最消耗人的问题：信息分散、真假难辨、进展断裂、支持难承接，以及普通人想帮却不知道怎么帮。平台通过结构化发布、轻量参与和关键记录上链，让救助更容易被看见、被承接，也更值得被信任。

---

## 🌍 项目背景

当前流浪动物救助高度依赖民间力量。很多真实救助都起于一个偶发瞬间：有人在路边看到受伤动物、捡到幼崽、遇到走失小狗，或者刷到一条快被淹没的领养帖。问题并不只是资源不够，更在于信息分散、真假难辨、进展断裂、支持难承接，以及普通人即使想帮一点，也常常不知道该怎么帮。

现实里，个人偶发救助者通常没有长期资源、没有可信背书，也没有能力独自扛完整个救助链路；长期救助者、小院和组织又本来就处于高压状态，不可能无条件接收所有偶发 case。RescueLink 希望先把需求表达、进展更新、凭证留痕、资源缺口、互助支持与后续承接整理进更清晰的结构里，让一次救助不再停留在碎片化求助。

---

## 💡 我们在做什么

它援 RescueLink 不是单纯的信息发布页，也不是传统募捐平台，而是一个面向流浪动物救助场景的互助支持工具。它通过结构化的需求表达、进展更新、凭证留痕、资源缺口展示、互助支持与后续承接机制，把分散的救助信息和可参与的帮助方式连接到同一个可信页面中。

---

## 🎯 核心价值

### 🆘 对个人偶发救助者
当你临时遇到需要帮助的动物时，可以快速发起需求、更新进展、上传凭证，并把一个原本散落在社交平台、私聊和截图里的救助过程整理成更清楚、更可追踪的页面。

### 🙌 对普通用户 / 助力人
当你想帮一点，但不知道怎么帮时，平台会优先呈现更真实、更紧急、更适合你参与的需求，让浏览、分享、认领、下单等轻量参与方式真正流向帮助。

### 🏡 对长期救助者 / 小院
通过认证主体主页和资源 Case，长期救助主体可以获得更稳定、更可信的长期补给入口，而不是一次次重复解释和零散求助。

### 🤝 对品牌 / 服务方
品牌资源、公益文创和服务支持可以更真实地连接具体需求，并形成可记录的公益影响路径。

## 🧩 产品结构

### 1. 🐶 救助 Case
用于承载一只具体动物或一次具体救助链路中的单次需求，例如医疗、驱虫、绝育、粮食、药品、陪诊、转运、临时寄养、开放领养、回访等。它是平台最核心的内容单元。

### 2. 📦 资源 Case
面向认证主体发布，用于承载长期、持续、开放式的补给型需求，例如猫粮、狗粮、猫砂、药品储备、旧被子、笼子、航空箱等。它不对应一次性、定额、与单只动物直接绑定的单次缺口。

### 3. 🏷️ 认证主体主页
长期救助个人、小院、团队或组织可拥有认证主页，用于展示基础信息、认证状态、城市、过往记录、当前资源 Case、正在进行的救助 Case、近期更新和长期接收说明。它不是泛社交主页，而是可信主体档案和长期资源入口。

### 4. 🗺️ 救助伙伴
“救助伙伴”用于承接认证主体主页、长期资源 Case、服务方、有偿寄养与救助地图。这里展示的是可以被查看和连接的长期伙伴网络，而不是默认会承接个案的承诺池。

### 5. ⭐ 助力中心
平台不把普通用户都变成重度救助者，而是把很多低门槛动作设计成可反馈、可累计、可继续流向真实需求的互助支持路径。浏览、分享、认领、下单、完成指南学习等行为都可以形成助力值与后续参与记录。

## 🔐 Trust & Record Mechanism

建立信任不是附属功能，而是产品核心。

RescueLink 的信任机制包括：

- **认证**：对长期救助主体进行认证和分级
- **记录**：需求、更新、签收、接管、领养、回访持续挂在同一链路下
- **凭证**：医疗、服务和关键支持完成后上传凭证或完成记录
- **存证**：对 Case 创建、关键更新、凭证哈希、兑换、交接、结案等做摘要上链

我们希望回应的是现实里真实存在的问题：骗捐、伪救助、断更、失联、资源滥用，以及支持去向不清。

## ✨ 轻量参与方式

用户不需要成为专业救助者，也可以参与：

- 浏览 Case 更新
- 分享 Case、生成海报、复制传播文案
- 认领资源、提交签收
- 购买合作商品或公益文创
- 学习救助指南、领养指南、寻宠指南
- 上传凭证、完成回访或交接等更深参与行为

这些行为会形成助力值与参与记录，但助力值**不直接等价现金，不可提现**。

## ⚖️ 合规边界

RescueLink 的定位不是公众募捐平台。当前方案明确：

- 不做公众募捐
- 不直接经手善款
- 不做资金归集和分账
- 积分余额和日常积分变动链下管理
- 关键兑换、核销、签收和重要支持结果做摘要上链
- 平台不替代医院、寄养方或领养方自身责任
- 服务方可展示，但平台不默认替其背书

## 🧭 How It Works

1. **发起需求**  
   发起方创建救助 Case 或由认证主体发布资源 Case，清楚表达对象、阶段、缺口与当前状态。

2. **持续更新**  
   救助进展、关键节点、服务完成情况、签收和交接都持续挂在同一条链路下。

3. **轻量参与**  
   普通用户可以通过浏览、分享、认领、下单、学习指南等低门槛动作参与支持。

4. **凭证留痕**  
   医疗、服务和关键支持完成后上传凭证或完成记录，减少“说不清”和“查不到”。

5. **关键记录上链**  
   对 Case 创建、重要更新、凭证哈希、交接和结案等关键节点进行摘要上链，提供可验证的可信记录。

6. **形成后续承接**  
   个案可以继续延伸到寄养、领养、回访和接管，避免一次救助在中途断裂。
